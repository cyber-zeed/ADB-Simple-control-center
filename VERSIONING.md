# Versioning

ADB Control Center uses semantic-style versioning while it is pre-1.0:

- `MAJOR`: breaking redesigns or incompatible project layout changes
- `MINOR`: new features or meaningful workflow additions
- `PATCH`: bug fixes, validation hardening, small UI improvements

The current release is `0.7.0`, a minor release because it adds persistent settings, cancellable long-operation UI, close-time Logcat protection, and GitHub Actions validation.
